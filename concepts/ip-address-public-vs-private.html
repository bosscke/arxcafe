IP Address (Public vs Private)

IP addresses identify devices on a network. Public IPs are globally reachable on the internet. Private IPs are only valid inside private networks (home, office, cloud VPC/VNet) and are not directly routable on the public internet.

Public vs Private at a Glance

Public IP

  • Globally unique and internet-routable
  • Assigned by ISPs or cloud providers
  • Needed for services exposed to the internet
  • Can be static or dynamic

Private IP

  • Not routable on the public internet
  • Safe to reuse across separate networks
  • Used inside LANs, VPNs, VPCs/VNets
  • Relies on NAT to reach the internet

RFC1918 Private IPv4 Ranges

10.0.0.0/8

16,777,216 addresses (large enterprises, clouds).

172.16.0.0/12

1,048,576 addresses (common in corp networks).

192.168.0.0/16

65,536 addresses (home routers default here).

Link-local: 169.254.0.0/16

Self-assigned when DHCP fails; not routed.

How NAT Bridges Private to Public

Outbound internet from a private network:
Client uses private IP (e.g., 192.168.1.10)
Packet hits NAT gateway/router
NAT rewrites source to public IP + port
Return traffic mapped back to internal host
Internal device never exposed directly

IPv4 vs IPv6 Context

IPv4

  • 32-bit address space (~4.3B addresses)
  • Exhaustion drives heavy NAT use
  • Private ranges defined by RFC1918

IPv6

  • 128-bit space (virtually inexhaustible)
  • Typically global unicast; NAT often unnecessary
  • ULA (fc00::/7) for private-like addressing

Quick Checks & Tips

Identify address type fast:

  • 10.x.x.x, 172.16-31.x.x, 192.168.x.x → private IPv4
  • 169.254.x.x → link-local (DHCP issue?)
  • Public IPv4: anything else allocated via RIR/ISP
  • IPv6: fc00::/7 (ULA) behaves like private

Operational tips:

  • Keep services on private IPs; front with load balancer or NAT
  • Use security groups/firewalls even for private ranges
  • Reserve/elastic IPs for stable public endpoints

Examples

Home network

Router public IP: 203.0.113.45 LAN: 192.168.1.0/24 Laptop: 192.168.1.20 Phone: 192.168.1.21 NAT: maps outbound to 203.0.113.45:random_port

Cloud VPC/VNet

VPC CIDR: 10.20.0.0/16 Private subnet: 10.20.1.0/24 App server: 10.20.1.15 (no public IP) NAT gateway public IP: 198.51.100.9 Load balancer public IP: 198.51.100.25 → routes to 10.20.1.15

Public IPs make a service reachable worldwide. Private IPs keep internal resources isolated and scalable. NAT and load balancers bridge the two worlds securely.

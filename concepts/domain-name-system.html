Domain Name System (DNS)

DNS is the internetâ€™s distributed phonebook. It turns human-friendly names (like www.example.com) into IP addresses that computers route to. DNS is hierarchical, cached everywhere, and designed for resilience and delegation.

DNS Hierarchy

Root

  • 13 logical root server clusters
  • Knows where TLD servers live
  • Returns referrals, not final answers

TLD (Top-Level Domains)

  • .com, .org, .net, country codes, and more
  • Delegates to authoritative servers for domains
  • Operated by registries (Verisign, PIR, etc.)

Authoritative Servers

  • Hold the zone file for your domain
  • Return the definitive records (A/AAAA/CNAME/...)
  • Configured via your DNS host or registrar

Recursive Resolvers

  • Do lookups for clients and cache results
  • Typically ISP resolvers or public (1.1.1.1, 8.8.8.8)
  • Shield authoritative servers from excessive load

Core Concepts

Zones & Delegation

Domains are split into zones that delegate downward via NS records.

TTL (Time To Live)

How long caches may hold an answer; lower before changes, higher for stability.

Propagation

Changes roll out as caches expire; not instant worldwide.

Anycast

DNS servers share one IP advertised from many locations for speed/resilience.

UDP & TCP

Most queries use UDP/53; TCP/53 used for larger responses, DNSSEC, zone transfers.

DNSSEC

Cryptographic signatures (RRSIG, DS) to prevent tampering in transit.

Common Record Types

A / AAAA

IPv4 and IPv6 addresses for hosts.

CNAME

Alias to another name; cannot coexist with other records on that name.

NS

Delegation to authoritative nameservers for the zone.

MX

Mail exchange servers with priority values.

TXT

Arbitrary text: SPF, DKIM, ownership verification.

SOA

Zone metadata: primary NS, contact, serial, refresh/retry timers.

Example: Zone Snippet

$ORIGIN example.com. $TTL 3600 @ IN SOA ns1.example.com. admin.example.com. ( 2025010101 ; serial 7200 ; refresh 3600 ; retry 1209600 ; expire 300 ) ; minimum IN NS ns1.example.com. IN NS ns2.example.com. www IN A 93.184.216.34 mail IN MX 10 mail.example.com. _spf IN TXT "v=spf1 include:_spf.example.com ~all"

Operating DNS Safely

Best practices:

  • Use at least two authoritative nameservers in different regions/providers
  • Lower TTLs before planned migrations; raise after stable
  • Enable DNSSEC if your registrar and host support it
  • Separate auth DNS from app hosting for blast radius reduction
  • Monitor for unexpected changes (records, NS, DS)

Troubleshooting Cheats

Check these first:

  • NS delegation matches registrar records
  • SOA serial increments after zone edits
  • Conflicting CNAME with other records on same name
  • Firewall blocking UDP/53 (and TCP/53 for large replies)
  • Stale caches: test with `dig @1.1.1.1` vs `dig @8.8.8.8`

DNS keeps the internet reachable by mapping names to addresses quickly and reliably. Knowing its hierarchy, caching, and records helps you plan changes confidently and debug issues fast.

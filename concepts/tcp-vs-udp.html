TCP vs UDP

Both TCP and UDP live at the transport layer (L4) and use ports to multiplex connections. TCP provides reliable, ordered, byte-stream delivery with congestion control. UDP is connectionless and best-effort: fast and lightweight, but with no guarantees.

Key Differences

TCP

  • Connection-oriented (3-way handshake)
  • Reliable delivery (ACKs, retransmissions)
  • Ordered byte stream
  • Flow + congestion control
  • Heavier header, more overhead

UDP

  • Connectionless (no handshake)
  • No delivery or ordering guarantees
  • Message-oriented (datagrams)
  • Very low overhead
  • App implements reliability if needed

When to Use Which

Use TCP when…

  • You need reliability and ordering (APIs, web, DBs)
  • Throughput matters more than minimal latency
  • Firewalls/proxies are common on the path

Use UDP when…

  • You need low latency and can tolerate loss
  • Real-time media (VoIP, video), gaming, telemetry
  • Custom reliability/FEC built at the app level

Behaviors & Mechanics

TCP Mechanics

  • Handshake: SYN → SYN-ACK → ACK
  • ACKs + retransmit on loss
  • Nagle’s algorithm, slow start, windowing
  • Head-of-line blocking

UDP Mechanics

  • No session state on network
  • Per-datagram size limited by path MTU
  • Loss/dup/out-of-order possible
  • Often paired with QUIC/DTLS for security/reliability

Common Use Cases

TCP Examples

HTTPS, REST APIs, SSH, SMTP/IMAP, database clients, file transfer (SFTP).

UDP Examples

DNS queries, VoIP/RTP, streaming, online gaming, metrics (StatsD), DHCP.

Code & Tools

Node.js TCP echo (server)

// tcp-echo.js const net = require('net'); net.createServer(s => s.pipe(s)).listen(9000);

Node.js UDP echo (server)

// udp-echo.js const dgram = require('dgram'); const sock = dgram.createSocket('udp4'); sock.on('message', (msg, rinfo) => { sock.send(msg, rinfo.port, rinfo.address); }); sock.bind(9001);

Windows: active TCP/UDP ports

PS> Get-NetTCPConnection -State Listen | Select-Object LocalAddress,LocalPort,OwningProcess PS> Get-NetUDPEndpoint | Select-Object LocalAddress,LocalPort,OwningProcess

Choose TCP for reliability and ordered delivery; choose UDP for low-latency, loss-tolerant applications. Many modern protocols (like QUIC) build reliability and security on top of UDP to get the best of both worlds.

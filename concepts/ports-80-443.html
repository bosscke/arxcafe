Ports (80, 443)

Network ports identify application endpoints on a host. Port 80 is the default for HTTP (unencrypted). Port 443 is the default for HTTPS (HTTP over TLS). Browsers omit the port when it’s the default, but you can specify them explicitly.

What 80 and 443 Mean

Port 80 (HTTP)

  • Unencrypted HTTP by default
  • Used for legacy or redirect to HTTPS
  • Often blocked inbound on hardened networks

Port 443 (HTTPS)

  • Encrypted via TLS (confidentiality, integrity)
  • Required for HSTS and modern browsers’ security features
  • Standard for public web traffic

URL Examples

http://example.com (implicit :80)
http://example.com:80 (explicit :80)
https://example.com (implicit :443)
https://example.com:443 (explicit :443)

Where Ports Fit (Layers)

Layer 4

Ports belong to TCP/UDP (transport). They multiplex many connections on one IP.

Layer 7

HTTP/HTTPS are application protocols carried over TCP (usually) on ports 80/443.

Firewalls/LB

Security groups and load balancers allow/forward specific ports to backends.

Common Inbound/Outbound Patterns

Inbound

  • Public sites listen on 443 (and maybe 80 → redirect)
  • Load balancer terminates TLS, forwards to private ports
  • WAF/CDN can sit in front of 443 for security/caching

Outbound

  • Clients egress to remote :80/:443 through NAT/proxy
  • Firewalls allow egress 443 by default in many orgs
  • Pin outbound destinations with allowlists if needed

Quick Examples

cURL with explicit ports

$ curl -I http://example.com:80/ $ curl -I https://example.com:443/

Windows: listening ports

PS> Get-NetTCPConnection -State Listen | Where-Object { $_.LocalPort -in 80,443 } | Select-Object LocalAddress,LocalPort,OwningProcess

Ops Tips & Pitfalls

What to watch:

  • Redirect port 80 → 443 for SEO and security
  • Open only required inbound ports on firewalls/NACLs
  • Terminate TLS at LB or at app; be clear on trust boundaries
  • Certificate renewals (e.g., ACME/Let’s Encrypt) must pass on 80 or 443

80 and 443 are the web’s front doors. Use 443 for encrypted traffic, keep 80 for redirects or ACME challenges, and enforce least privilege at the network edge.

Headers and Body

Headers and Body are the two main components of HTTP requests and responses. Headers are metadata key-value pairs that provide information about the request/response (content type, authentication, caching, etc.), while the Body (also called payload) contains the actual data being transmitted. Together, they enable rich communication between clients and servers, allowing flexible data formats, authentication, and control of how data is processed and cached.

Headers vs Body

Headers

  • Purpose: Metadata about request/response
  • Format: Key-Value pairs (e.g., "Content-Type: application/json")
  • Location: Always included, before body
  • Required: Some are mandatory (Host, User-Agent)
  • Visible: Easily inspected in dev tools
  • Size: Typically small (few KB)
  • Examples: Content-Type, Authorization, Cookie, Cache-Control

Body

  • Purpose: Actual data being transmitted
  • Format: Varies (JSON, XML, HTML, binary, form data, etc.)
  • Location: After headers, optional for some requests
  • Required: Not needed for GET requests
  • Visible: May be text or binary
  • Size: Can be large (MB to GB)
  • Examples: JSON data, HTML page, image bytes, file upload

Common Request Headers

Essential Request Headers:

Authentication & Identity

Authorization: Bearer token, credentials

Cookie: Session data

User-Agent: Client info (browser, OS)

Host: Domain being requested

Content & Format

Content-Type: Format of body (application/json)

Content-Length: Size of body in bytes

Accept: Formats client understands

Accept-Encoding: Compression support (gzip)

Caching & Control

Cache-Control: How to cache response

If-Modified-Since: Conditional request

ETag: Resource version identifier

Pragma: Legacy cache control

Security & Origin

Origin: Where request came from

Referer: Previous page URL

X-Requested-With: AJAX identifier

X-CSRF-Token: CSRF protection

Common Response Headers

Caching Headers

  • Cache-Control: How long to cache (max-age=3600)
  • ETag: Version tag for validation
  • Last-Modified: When resource last changed
  • Expires: Expiration date/time
  • Vary: Cache key dependencies

Content Headers

  • Content-Type: Type of response body
  • Content-Length: Size of response body
  • Content-Encoding: Compression (gzip, deflate)
  • Content-Disposition: Download vs display
  • Content-Range: Partial content info

Security Headers

  • Set-Cookie: Create session cookies
  • Strict-Transport-Security: Force HTTPS
  • X-Content-Type-Options: Prevent MIME sniffing
  • X-Frame-Options: Clickjacking protection
  • Content-Security-Policy: Script injection protection

Server Info Headers

  • Server: Server software type
  • Access-Control-Allow-Origin: CORS permission
  • Location: Redirect destination
  • Retry-After: Retry timing (503 responses)
  • Age: Cache age in seconds

Request Body Formats

JSON (Most Common)

Use: APIs, data exchange

Content-Type: application/json

Example:

{"name": "John", "age": 30}

Form Data (URL-Encoded)

Use: HTML forms, login

Content-Type: application/x-www-form-urlencoded

Example:

name=John&age=30

Multipart Form (File Upload)

Use: File uploads, mixed data

Content-Type: multipart/form-data

Example:

--boundary\r\nContent-Disposition: form-data; name="file"\r\n\r\n[bytes]

XML

Use: Legacy APIs, SOAP

Content-Type: application/xml

Example:

<user><name>John</name></user>

HTTP Request Example

POST Request with Headers and Body:

POST /api/users HTTP/1.1
Host: api.example.com
Content-Type: application/json
Authorization: Bearer token123

// BODY STARTS HERE
{"name":"John","email":"john@example.com"}

HTTP Response Example

200 OK Response with Headers and Body:

HTTP/1.1 200 OK
Content-Type: application/json
Cache-Control: max-age=3600
Set-Cookie: sessionId=xyz789; HttpOnly

// BODY STARTS HERE
{"id":1,"name":"John","email":"john@example.com"}

Code Examples

Working with Headers and Body in JavaScript:

// Fetch API - Setting Headers and Body
fetch('https://api.example.com/users', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'Authorization': 'Bearer token123'
    },
    body: JSON.stringify({name: 'John', age: 30})
}).then(res => res.json()).then(data => console.log(data));

// Express.js - Reading and Setting Headers
app.post('/api/users', (req, res) => {
    const auth = req.headers['authorization'];
    res.set('Cache-Control', 'max-age=3600');
    res.json({id: 1, ...req.body});
});

When to Use Headers vs Body

Use Headers For

  • Content type and encoding
  • Authentication tokens
  • Cookies and sessions
  • Cache control directives
  • Compression negotiation
  • CORS permissions
  • Request/response metadata

Use Body For

  • User input (forms, JSON)
  • Data to be created/updated
  • File uploads
  • Large datasets
  • Sensitive information
  • Complex nested structures
  • Media files (images, videos)

Common Mistakes

Headers and Body Anti-Patterns

  • Forgetting Content-Type: Server can't parse body correctly
  • Wrong Content-Type: Sending JSON but declaring text/html
  • Large Headers: Oversized headers slow down requests
  • Putting Data in Headers: Headers have size limits (~8KB)
  • Not Using HTTPS: Headers/body sent in plain text
  • Exposed Tokens in Headers: Visible in network logs
  • Ignoring Cache Headers: Unnecessary duplicate requests
  • Empty Bodies on POST: Intent unclear to server

Best Practices

Working with Headers and Body Effectively:

  • Always Set Content-Type: Be explicit about body format
  • Use Appropriate HTTP Methods: GET has no body, POST/PUT do
  • Validate Headers: Check required headers are present
  • Parse Body Safely: Handle parsing errors gracefully
  • Implement Caching Headers: Reduce server load with Cache-Control
  • Use Compression: Set Accept-Encoding for smaller payloads
  • Secure Headers: Use HTTPS, set secure cookies
  • Document Headers: List required headers in API docs
  • Use JSON for APIs: Standard, readable, widely supported
  • Log Safely: Don't log sensitive headers (Authorization)

Headers and Body work together to enable sophisticated HTTP communication. Headers provide metadata and control, while the body carries the actual data. Understanding both is essential for building APIs, debugging web requests, and implementing proper security and performance practices.

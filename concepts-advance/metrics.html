Metrics

Metrics are quantifiable measurements of system performance, user behavior, and business outcomes. Metrics provide numerical data about how systems are functioning, enabling data-driven decisions about performance optimization, capacity planning, and business strategy. Effective metrics track what matters most: performance, reliability, user experience, and business goals.

Core Concepts

Metric Types

Different ways to measure system behavior.

  • Gauge: Single value at point in time
  • Counter: Monotonically increasing value
  • Histogram: Distribution of values
  • Summary: Quantiles over time windows
  • Rate: Change per time unit

Cardinality

Number of unique label combinations in metrics.

  • High cardinality can cause storage issues
  • Limit dynamic labels (user IDs, request IDs)
  • Use aggregation to reduce cardinality
  • Monitor cardinality explosion
  • Design labels carefully

Sampling & Aggregation

Techniques for managing metric data volume.

  • Sampling: collect subset of data
  • Aggregation: combine raw metrics
  • Reduce storage and network costs
  • Trade precision for efficiency
  • Time-series databases

Retention Policies

How long to keep metric data.

  • High-resolution data: hours/days
  • Medium resolution: weeks
  • Downsampling over time
  • Archive for long-term analysis
  • Cost vs retention trade-off

Categories of Metrics

RED Metrics

Request-focused metrics for services.

  • Rate: Requests per second
  • Errors: Failed requests
  • Duration: Latency/response time
  • User-centric view
  • Great for APIs and services

USE Metrics

Resource-focused metrics for infrastructure.

  • Utilization: Resource in use
  • Saturation: Queued requests
  • Errors: Device errors
  • CPU, memory, disk, network
  • Great for servers and resources

Business Metrics

Metrics aligned with business goals.

  • Revenue, conversion rate
  • User engagement
  • Customer satisfaction
  • Churn rate
  • Directly tied to value

System Metrics

Low-level system behavior metrics.

  • CPU, memory, disk usage
  • Network I/O
  • Context switches
  • Page faults
  • Infrastructure foundation

Key Application Metrics

Latency

Response time measurement.

  • p50, p95, p99 percentiles
  • Average vs tail latency
  • Microseconds to seconds
  • User experience impact
  • Histogram metric type

Error Rate

Failed requests percentage.

  • Total errors / total requests
  • Error types breakdown
  • HTTP status codes
  • Exception counts
  • Gauge or counter

Throughput

Requests processed per unit time.

  • Requests per second
  • Transactions per minute
  • Capacity indicator
  • Growth tracking
  • Counter metric type

Saturation

How full systems are.

  • Queue length
  • Connection pool usage
  • CPU time waiting
  • Resource exhaustion indicator
  • Gauge metric type

Advantages

Benefits

  • Performance Visibility: Understand system behavior
  • Capacity Planning: Predict resource needs
  • Problem Detection: Identify issues early
  • Data-Driven Decisions: Optimize based on data
  • Scalability Insights: Know when to scale
  • Cost Optimization: Reduce wasteful spending
  • SLA Compliance: Track agreement commitments
  • User Experience: Monitor perceived performance

Challenges

Drawbacks

  • Storage Costs: Metrics data grows exponentially
  • Cardinality Explosion: High-cardinality labels cause issues
  • Data Quality: Garbage in, garbage out
  • Overhead: Metric collection impacts performance
  • Alert Fatigue: Too many alerts reduce effectiveness
  • Choosing Right Metrics: What to measure is hard
  • Correlation vs Causation: Metrics don't explain why
  • Infrastructure Complexity: Setup and maintenance required

Best Practices

What to Measure

  • Focus on business outcomes
  • Track user experience metrics
  • Monitor resource utilization
  • Alert on SLA violations
  • Avoid measuring everything

Metric Design

  • Use consistent naming conventions
  • Careful label selection
  • Avoid high cardinality labels
  • Include meaningful dimensions
  • Document metric meaning

Collection & Storage

  • Use efficient instrumentation
  • Implement sampling strategically
  • Set appropriate retention periods
  • Use time-series databases
  • Monitor storage costs

Analysis & Action

  • Create meaningful dashboards
  • Set context-aware thresholds
  • Use metrics for debugging
  • Track trends over time
  • Share insights with teams

Metrics vs Logs vs Traces

Metrics

Aggregated numerical data.

  • Quantitative measurements
  • Time-series data
  • Answers "how much?"
  • Low storage
  • Real-time dashboards

Logs

Discrete text events.

  • Detailed event records
  • Text or structured
  • Answers "what happened?"
  • High storage
  • Post-mortem analysis

Traces

Request journey across systems.

  • Distributed transactions
  • End-to-end visibility
  • Answers "how did this happen?"
  • Medium storage
  • Performance debugging

Combined Approach

  • Metrics for detection
  • Logs for diagnosis
  • Traces for understanding
  • Three pillars of observability
  • Together = complete visibility

Establishing Metrics

Step 1: Define Goals

  • What are business objectives?
  • What matters to users?
  • What indicates health?
  • What prevents incidents?
  • Align with SLOs

Step 2: Choose Metrics

  • RED for services
  • USE for resources
  • Business metrics
  • Avoid vanity metrics
  • Start small, grow thoughtfully

Step 3: Instrument Code

  • Use metric libraries
  • Instrument latency
  • Count requests/errors
  • Track resource usage
  • Minimal overhead

Step 4: Analyze & Act

  • Create dashboards
  • Set meaningful alerts
  • Identify trends
  • Optimize based on data
  • Iterate continuously

Metrics are the foundation of modern system observability. By collecting and analyzing the right metrics, you gain visibility into your systems, understand user experience, and make data-driven decisions about optimization and resource allocation. Start simple, measure what matters, and build from there.

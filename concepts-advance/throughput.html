Throughput

Throughput is the amount of data, requests, or transactions a system can process in a given period of time. It measures the system's capacity to handle workload and is typically expressed as requests per second (RPS), transactions per second (TPS), or bits per second (bps). High throughput means the system can handle many concurrent operations efficiently.

Core Concepts

Definition

The rate at which a system processes work over time.

  • Volume per time unit
  • System capacity measure
  • Sustained performance
  • Overall efficiency indicator
  • Scalability metric

Common Measurements

Different ways to measure throughput based on context.

  • Requests per second (RPS)
  • Transactions per second (TPS)
  • Queries per second (QPS)
  • Bits/bytes per second (bps)
  • Messages per second (MPS)

Throughput vs Latency

Throughput

How much work gets done overall.

  • Volume-focused
  • Aggregate measure
  • System-level metric
  • Example: 10,000 requests/sec
  • Batch processing oriented

Latency

How fast individual requests complete.

  • Speed-focused
  • Individual measure
  • Request-level metric
  • Example: 50ms per request
  • Real-time oriented

The Relationship:

You can have high throughput with high latency (processing many slow requests), or low throughput with low latency (processing few fast requests). Ideal systems optimize both: high throughput AND low latency.

Factors Affecting Throughput

Bandwidth

Network capacity limits data transfer rate.

  • Physical connection speed
  • Network congestion
  • Protocol overhead
  • Compression efficiency

Processing Capacity

CPU and memory available for computation.

  • CPU cores and speed
  • Memory availability
  • Algorithm efficiency
  • I/O performance

Concurrency

Number of parallel operations possible.

  • Thread/process limits
  • Connection pooling
  • Async processing
  • Resource contention

Bottlenecks

Slowest component limits overall throughput.

  • Database queries
  • Disk I/O
  • External API calls
  • Lock contention

How to Increase Throughput

Optimization Strategies:

  • Horizontal Scaling: Add more servers to distribute load
  • Vertical Scaling: Upgrade hardware (CPU, RAM, SSD)
  • Load Balancing: Distribute requests evenly across servers
  • Caching: Reduce redundant processing (Redis, Memcached)
  • Async Processing: Use message queues for background tasks
  • Database Optimization: Indexes, query optimization, read replicas
  • Connection Pooling: Reuse database connections
  • CDN Usage: Offload static content delivery
  • Compression: Reduce data transfer size
  • Batch Processing: Process multiple items together
  • Remove Bottlenecks: Profile and optimize slow code paths
  • Parallel Processing: Multi-threading and concurrency

Common Bottlenecks

What Limits Throughput:

  • Database Locks: Concurrent write operations blocking each other
  • N+1 Queries: Excessive database round trips
  • Single-threaded Processing: Not utilizing available CPU cores
  • Synchronous I/O: Blocking operations preventing parallelism
  • Memory Leaks: Gradual performance degradation
  • Network Saturation: Bandwidth limits reached
  • CPU Saturation: Processing power maxed out
  • Disk I/O Limits: Slow hard drives or excessive writes
  • External Dependencies: Slow third-party APIs

Best Practices

Optimizing for High Throughput:

  • Profile Before Optimizing: Measure to find actual bottlenecks
  • Design for Concurrency: Use async/await, non-blocking I/O
  • Batch Operations: Process multiple items in single database call
  • Use Connection Pools: Reuse expensive connections
  • Implement Caching: Cache at multiple layers (CDN, app, database)
  • Monitor Continuously: Track throughput trends over time
  • Load Test Regularly: Verify system can handle expected traffic
  • Set SLOs: Define acceptable throughput targets
  • Optimize Hot Paths: Focus on most-used code paths
  • Use Appropriate Data Structures: Choose efficient algorithms

Capacity Planning

Planning for Growth:

  • Estimate Peak Load: Calculate maximum expected traffic
  • Add Buffer: Plan for 2-3x expected load
  • Test Limits: Load test to find breaking points
  • Monitor Trends: Track growth over time
  • Auto-Scaling: Automatically adjust capacity
  • Cost Optimization: Balance performance vs infrastructure cost
  • Redundancy: Ensure no single point of failure

Throughput is a critical measure of system capacity. While latency focuses on individual request speed, throughput determines how many users your system can serve simultaneously. Optimizing throughput requires understanding bottlenecks, efficient resource utilization, and strategic scaling.

Secrets Management

Secrets Management is the secure storage, access, and rotation of sensitive credentials like API keys, database passwords, encryption keys, and tokens. Proper secrets management prevents hardcoding credentials in code, enables secure sharing across environments, and provides audit trails for compliance. It's essential for security, preventing data breaches, and maintaining operational control.

Core Concepts

What Are Secrets

Types of sensitive data.

  • API keys and tokens
  • Database passwords
  • Encryption keys
  • SSH private keys
  • OAuth credentials

Secret Lifecycle

Managing secrets over time.

  • Generation/Creation
  • Storage (encrypted)
  • Access (controlled)
  • Rotation (periodic)
  • Revocation (emergency)

The Problem

Why secrets matter.

  • Hardcoding = instant leak risk
  • Source code accessible to devs
  • Git history persists forever
  • Environment variables exposed
  • Lost credentials = breaches

Core Principles

Best practice foundations.

  • Never commit to source control
  • Encrypt at rest always
  • Encrypt in transit (TLS)
  • Principle of least privilege
  • Audit all access

Secrets Storage Methods

Environment Variables

Simple per-process secrets.

  • Stored in process memory
  • Not in source code
  • Set per environment
  • Good for containers
  • Limited audit trail

.env Files

Local development secrets.

  • .env in .gitignore
  • Local development only
  • Easy to use
  • Human readable
  • Never in production

Secrets Vaults

Centralized secret storage.

  • HashiCorp Vault
  • AWS Secrets Manager
  • Azure Key Vault
  • Encrypted storage
  • Access control + audit

Kubernetes Secrets

Container orchestration secrets.

  • K8s native Secret objects
  • Base64 encoded (not encrypted)
  • Mount as files/env vars
  • Use etcd encryption
  • RBAC access control

Secret Access Patterns

Direct Access

Application requests secrets.

  • App authenticates to vault
  • Requests specific secret
  • Vault returns decrypted
  • App uses in memory
  • Requires vault SDK/API

Injected Access

Secrets provided at runtime.

  • Sidecar injects secrets
  • Pod environment variables
  • Mounted files
  • No app SDK needed
  • K8s native approach

Dynamic Secrets

Temporary credentials.

  • Generated on demand
  • Short TTL (time-to-live)
  • Auto-revoked at expiry
  • DB credentials example
  • Reduced breach impact

Service Accounts

Machine-to-machine auth.

  • Service identity
  • Long-lived tokens
  • API keys/certs
  • Scoped permissions
  • Audit logged

Secret Rotation

Rotation Strategies

Keeping secrets fresh.

  • Scheduled rotation (monthly)
  • On-demand rotation
  • Automatic rotation
  • Zero-downtime rotation
  • Emergency rotation

Rotation Process

Safe secret updates.

  • Generate new secret
  • Validate new secret works
  • Update in vault
  • Apps fetch updated secret
  • Revoke old secret

Database Credentials

Rotating DB access.

  • Create new user/password
  • Update connection strings
  • Test new credentials
  • Migrate active connections
  • Delete old user

Challenges

Rotation complications.

  • Coordinating updates
  • Long-lived connections
  • Legacy system incompatibility
  • Testing requirements
  • Audit trail complexity

Industry-standard vault.

  • Secret engines

AWS Secrets Manager

AWS native solution.

  • IAM auth control

Azure Key Vault

Microsoft cloud secrets.

GCP secrets platform.

Advantages

Benefits

  • Security: No hardcoded credentials
  • Breach Prevention: Limits credential exposure
  • Rotation: Regular key cycling
  • Audit Trail: Who accessed what when
  • Compliance: Meets regulatory requirements
  • Zero Trust: Verifiable access control
  • Dynamic Secrets: Temporary credentials
  • Emergency Revocation: Quick breach response

Challenges

Drawbacks

  • Complexity: Multiple systems to manage
  • Performance: Vault access adds latency
  • Dependency: Vault outage = app issues
  • Development Friction: Extra setup needed
  • Bootstrap Problem: How do apps auth to vault?
  • Legacy Systems: Hard to integrate
  • Cost: Managed vault services charge
  • Learning Curve: Team training needed

Best Practices

Storage Security

  • Use dedicated vault
  • Encryption at rest (AES-256)
  • Encryption in transit (TLS)
  • High availability setup
  • Regular backups

Access Control

  • Principle of least privilege
  • RBAC for all access
  • MFA for human access
  • Service account auth
  • Regular permission audits

Operational Security

  • Never log secrets
  • No hardcoding anywhere
  • Automated rotation
  • Breach response plan
  • Regular scanning for secrets

Audit & Monitoring

  • Log all secret access
  • Alert on anomalies
  • Regular access reviews
  • Compliance audits
  • Retention policies

Secret Scanning in Development

Pre-commit Hooks

Prevent commits with secrets.

  • Run before commit
  • Scan for patterns
  • Detect API keys
  • Detect passwords
  • Block suspicious commits

CI/CD Scanning

Automated secret detection.

  • Scan pull requests
  • GitHub Advanced Security
  • GitLab SAST
  • Third-party tools
  • Fail pipeline if found

Secret Rotation Tools

Automated credential updates.

  • Detect compromised secrets
  • Auto-rotate immediately
  • Update applications
  • Notify teams
  • Audit trail

Repository Cleanup

Historical secret removal.

  • Scan git history
  • Find old secrets
  • Force push to remove
  • Notify developers
  • Rotate found secrets

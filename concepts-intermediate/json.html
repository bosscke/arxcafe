JSON (JavaScript Object Notation)

JSON is a lightweight data-interchange format based on key/value pairs and arrays. It is language-agnostic, human-readable, and widely used for APIs, configs, and messaging.

Core Rules

Data Types

Objects, arrays, strings, numbers, booleans, null. No comments.

Strings

Must use double quotes; escape with backslash (e.g., \n, \" ).

Numbers

No leading zeros; no NaN/Infinity; decimals allowed.

Objects & Arrays

Objects are unordered key/value; arrays are ordered lists.

Examples

Valid JSON Object

{ "id": 1, "name": "Ada Lovelace", "skills": ["math", "programming"], "active": true, "meta": { "team": "platform", "location": null } }

Common Invalid JSON

// invalid because of single quotes and trailing comma { 'id': 1, "name": "Ada", "skills": ["math", "programming",], }

Parsing & Stringify (JavaScript)

const obj = JSON.parse('{"id":1,"name":"Ada"}'); const s = JSON.stringify(obj, null, 2); // pretty print

Parsing (Python)

import json raw = '{"id":1,"name":"Ada"}' data = json.loads(raw) json_text = json.dumps(data, indent=2)

Tips

Validation

Use JSON Schema to validate shape, types, and required fields.

Encoding

UTF-8 by default; ensure correct content-type headers.

Security

Do not eval JSON. Enforce size limits to prevent abuse.

APIs

Use consistent casing (snake_case or camelCase) and versioning.

Common Pitfalls

Avoid these:

  • Trailing commas or single quotes
  • Comments in JSON (use JSONC if needed)
  • Storing NaN/Infinity or dates without clear format
  • Mixing tabs/spaces in content strings improperly escaped

JSON is simple and ubiquitous. Keep it valid, validated, and versioned to ensure reliable data interchange.

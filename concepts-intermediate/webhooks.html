Webhooks

Webhooks are outbound HTTP callbacks that notify subscribers when events occur. They push data instead of polling, but require careful reliability, security, and observability.

Core Concepts

Subscription

Client registers a URL and secrets/headers for verification.

Delivery

Provider POSTs JSON payloads; expects 2xx. Retries on failure.

Verification

HMAC signatures or shared secrets to confirm authenticity.

Idempotency

Events may deliver more than once; receivers must dedupe.

Example: Sending & Receiving

Provider (Node.js) — send webhook

const crypto = require('crypto'); const fetch = require('node-fetch'); async function sendWebhook(targetUrl, secret, event) { const body = JSON.stringify(event); const sig = crypto .createHmac('sha256', secret) .update(body) .digest('hex'); const res = await fetch(targetUrl, { method: 'POST', headers: { 'Content-Type': 'application/json', 'X-Signature': sig, 'X-Event-Type': event.type }, body }); return res.status; }

Receiver (Node.js) — verify webhook

const express = require('express'); const crypto = require('crypto'); const app = express(); app.use(express.json({ verify: (req, res, buf) => { req.rawBody = buf; }})); function verify(req, secret) { const signature = req.headers['x-signature']; const expected = crypto.createHmac('sha256', secret).update(req.rawBody).digest('hex'); return crypto.timingSafeEqual(Buffer.from(signature||'', 'hex'), Buffer.from(expected, 'hex')); } app.post('/webhooks/order-created', (req, res) => { if (!verify(req, process.env.WH_SECRET || 'dev-secret')) { return res.status(401).json({ error: 'invalid signature' }); } // idempotency: check event.id already processed console.log('received event', req.body.id); res.sendStatus(200); }); app.listen(4005);

Retry with Backoff (concept)

maxAttempts = 6 backoff = [1s, 2s, 4s, 8s, 16s, 32s] retry only on 5xx/timeout; drop on 2xx/410; stop on 4xx (bad signature) DLQ failed deliveries for later replay

Tips

Security

HMAC signatures, HTTPS, and allowlist of source IPs if possible.

Idempotency

Include event IDs; receivers store processed IDs to dedupe.

Observability

Log deliveries, measure latency, surface success/fail rates to clients.

Self-service

Provide test delivery, replay, pause/resume, and delivery history.

Best Practices

Do this:

  • Send clear event types and schema versions
  • Retry with backoff; stop after a sane limit
  • Return 2xx quickly; process heavy work async
  • Document signature verification steps
  • Offer replay with idempotent events

Common Pitfalls

Avoid these:

  • Skipping signature checks (spoofing risk)
  • Assuming exactly-once delivery
  • Long-running processing inside the webhook response
  • No retry/backoff or dead-letter handling

Webhooks turn your platform into an event source. Secure them, make them reliable, and give consumers the tools to debug and replay deliveries.
